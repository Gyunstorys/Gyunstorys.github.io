2. Code

2.1 HashingTF

Spark사용 중 HashingTF를 사용하여 TF*IDF 연산을 하려고 확인 중 평소 TF 계산과 다르게 연산 되는 걸 목격했다.
평소 TF연산은 해당하는 Term에 따라 TF연산이 되었고 10차원이라고하면 Total Term Frequency가 높은 순으로 10개 vocabulary를 생성 하여 문서 별 Vector를 생성 하였다.
그와 같이 HashingTF도 동일 한 로직으로 해당하는 단어만 빨리 찾기위해 Hashing 함수를 통하여 숫자로 변환 시켜놓은 것인 줄 알았다.

그러나 실행 결과는 달랐다.

예시(차원을 5로 하였을 때)

Term TF(termId1,termId2)(빈도1,빈도2)
[나,는] (1,2)(1,1)
[안녕,하] (3)(2)
[나,는,안녕,하] (1,2,3)(1,1,2)

안녕,하의 경우 당연 히 다른 Term으로 분리 되어 카운팅 되어야 한다고 생각했지만 합쳐서 연산되었다.
곰곰히 생각을 해보니 HashingTF의 경우 Hash 충돌을 이용하여 차원을 만드는 것 같았다.
10개의 차원으로 한다고 하면 Term이 어디 Hash에 들어가는 지 확인하여 Term Frequency를 만들었다.
예시

Hash Term
1 나
2 는
3 안녕,하
